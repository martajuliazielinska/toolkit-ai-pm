# Tworzenie User Story

## 📋 Co robisz?
Tworzysz User Story w standardzie Jira dla backlogu produktu.

## 📂 Gdzie szukasz danych?
1. `/discovery/user-research/` - research użytkowników
2. `/backlog/` - istniejące US do referencji
3. `/strategy/product-vision.md` - kontekst produktu
4. `/strategy/roadmap.md` - timeline i priorytety

## 🧠 Jak robisz?
Użyj: **skill:user-story-template-jira**

## 📋 Output powinien zawierać:
1. Context - business problem
2. Objectives & User Story
3. Benefits
4. KPI & Cost-Benefit
5. Scope (In/Out/Stakeholders/Timeline)
6. NFR (Usability, Systems, Tests, Accessibility)
7. Acceptance Criteria (Gherkin format)

## 💾 Gdzie zapisujesz wynik?
→ `/backlog/[feature-name].md`

## 🔗 Powiązane:
- skill:user-story-template-jira
- /discovery/user-research/
