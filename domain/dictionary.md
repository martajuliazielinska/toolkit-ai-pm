# Domain Dictionary - Family Wallet

## Financial Consultation Domain

### Core Concepts

**Advisor** - External financial expert from partner firm. Has read-only access to user-shared financial data. Provides recommendations during consultation.

**Consultation** - Scheduled meeting between User and Advisor. Has statuses: BOOKED, CONFIRMED, COMPLETED, CANCELLED_BY_USER, CANCELLED_BY_ADVISOR.

**Financial Report** - User-generated snapshot of financial data they choose to share. Not live data. User controls scope (e.g., "last 3 months expenses only").

**Advisor Panel** - Separate web application for advisors. Read-only access to:
- User's shared Financial Report
- Consultation history
- Task to write summary and recommendations

### Data Models

**Consultation Table:**
- consultation_id (PK)
- user_id (FK)
- advisor_id (FK)
- scheduled_time (datetime)
- status (enum: BOOKED, CONFIRMED, COMPLETED, CANCELLED_BY_USER, CANCELLED_BY_ADVISOR)
- video_conference_url (generated by Zoom/Whereby API)
- payment_id (FK to one-time charge)
- summary_observations (TEXT)
- summary_recommendations (TEXT)

**Advisor Table:**
- advisor_id (PK)
- name (TEXT)
- specialization (TEXT)
- hourly_rate (DECIMAL)
- rating_average (DECIMAL, calculated from post-consultation ratings)
- status (active/inactive)

### Key Risks Identified

- **Risk 1 (Blocker):** Live vs Snapshot access model for user financial data. GDPR/security decision pending.
- **Risk 2 (High):** Cancellation and refund logic undefined. Business analysis pending.
- **Technical Challenge:** Payment module needs extension for one-time service charges (not just subscriptions).
- **Technical Challenge:** Video conferencing API integration (cost research needed).
